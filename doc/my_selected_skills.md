## Hướng Dẫn Sử Dụng Chi Tiết

### 1. @brainstorming
*   **Mục đích**: Chuyển đổi ý tưởng sơ khai thành kế hoạch chi tiết.
*   **Cách dùng**:
    > "Dùng **@brainstorming** để lên kế hoạch cho tính năng đăng nhập mới."

### 2. @clean-code
*   **Mục đích**: Refactor code để dễ đọc, dễ bảo trì.
*   **Cách dùng**:
    > "Hãy review file `utils.js` và áp dụng **@clean-code**."

### 3. @architecture
*   **Mục đích**: Đưa ra quyết định về cấu trúc dự án hoặc công nghệ.
*   **Cách dùng**:
    > "Sử dụng **@architecture** để thiết kế database cho hệ thống e-commerce."

### 4. @security-audit
*   **Mục đích**: Tìm lỗ hổng bảo mật trong code hoặc cấu hình.
*   **Cách dùng**:
    > "Chạy **@security-audit** trên thư mục `controllers/` để tìm lỗi injection."

### 5. @prompt-engineer
*   **Mục đích**: Cải thiện câu prompt của bạn để AI hiểu rõ hơn.
*   **Cách dùng**:
    > "Dùng **@prompt-engineer** để tối ưu câu hỏi về cách fix lỗi Docker."

### 6. @git-commit
*   **Mục đích**: Tạo message commit đúng chuẩn (feat, fix, chore...).
*   **Cách dùng**:
    > "Tôi vừa sửa lỗi login, hãy dùng **@git-commit** để tạo message."

### 7. @readme-generator
*   **Mục đích**: Viết tài liệu giới thiệu dự án đầy đủ.
*   **Cách dùng**:
    > "Tạo file README cho dự án này bằng **@readme-generator**."
